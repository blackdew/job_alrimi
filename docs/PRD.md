# 남해군 일자리·빈집 정보 알림 앱 PRD (Product Requirement Document)

> [!NOTE]
> 본 문서는 남해군 주민 및 예비 이주민을 위한 '지역 밀착형 생활 정보 플랫폼' 구축을 위한 기획서입니다.

## 1. 개요 및 추진 배경 (Overview & Background)

### 1.1 분석 배경 (Problem Statement)
*   **Web의 한계**: 기존 지자체 홈페이지(Web) 기반 정보 제공은 모바일 환경 변화에 뒤처짐.
*   **정보 비대칭성**: '정보의 비대칭성'과 '높은 탐색 비용'으로 주민들이 양질의 일자리/정책 혜택을 놓침.
*   **이주 장벽**: 외부(도시권) 예비 이주민들이 현지 주거(빈집) 정보를 얻기 힘들어 유입이 제한됨.

### 1.2 앱의 정의 (Product Definition)
*   **지속적 진화 (Agile)**: 완성된 시스템이 아닌, 주민 피드백을 통해 성장하는 플랫폼.
*   **핵심 가치**: 복잡한 검색 없이 **"알림(Push) 받고 → 전화(Call) 거는"** 직관적 프로세스.

### 1.3 수집 대상 (Target Data Sources)
본 서비스는 아래의 공신력 있는 웹사이트의 데이터를 실시간으로 수집하여 제공합니다.
*   **일자리 정보**:
    1.  남해군청 유관기관 소식 (새올): `.../modules/saeol/gosi.do...`
    2.  남해군청 구인구직 게시판: `.../portal/board/List.do...`
    3.  경남 워크넷 (남해군): `gyeongnam.work.go.kr/namhae`
*   **빈집 정보**:
    1.  남해군 귀농귀촌지원센터: `refarm.namhae.go.kr`
    2.  그린대로 (농촌 빈집은행): `greendaero.go.kr`

## 2. 타겟 사용자 및 효능감 분석 (Target Audience & Value Proposition)

### 2.1 [3040 세대] 탐색 비용 '0'의 기회 선점 (Speed & Efficiency)
*   **Pain Point**: 생업으로 바빠 홈페이지를 매번 확인할 시간 부족.
*   **Solution**: **실시간 푸시 알림 (0.1초의 승부)**.
*   **Value**: 원하는 키워드(사무직, 전세 등) 등록 시 즉시 알림 → 경쟁 우위 선점.

### 2.2 [50대] 재취업과 노후 준비의 현실적 무기 (Core User)
*   **Pain Point**: 정보 욕구는 높으나 노안 등 신체적 변화 시작.
*   **Solution**: **정보의 질(Quality) + 가독성(Readability)**.
*   **Value**: 숙련직/관리직 등 알짜 정보 제공 + 눈이 편안한 UI. 구전 마케팅의 핵심 주체.

### 2.3 [6070 세대] 디지털 소외 해결 (Accessibility)
*   **Pain Point**: 모바일 사용은 가능하나 복잡한 UI/작은 글씨는 장벽.
*   **Solution**: **직관적 행동 유도 (Push to Call)**.
*   **Value**: 검색 과정 제거. 알림 → 큰 전화 버튼 클릭 → 연결. "나도 할 수 있다"는 자존감 회복.

### 2.4 [New: 이주민] 이주 장벽 낮추는 가이드 (Migration)
*   **Pain Point**: 현지에 가지 않으면 정보 획득 불가능.
*   **Solution**: **'Work & Stay' 통합 알림**.
*   **Value**: 타지에서도 남해의 빈집/일자리 정보 실시간 수신 → 심리적/실질적 이주 장벽 해소.

## 3. 이해관계자 가치 (Stakeholder Value)

### 3.1 공급자 (지자체/구인처) 효능감
*   **매칭 혁신**: 30~50대 핵심 인력 및 외부 유입으로 지원자 풀(Pool) 확대.
*   **인구 유입**: 주거+일자리 패키지 정보로 귀촌 실제 전입률(Conversion) 증대.
*   **행정 효율**: 홍보 문자 예산 절감 → 앱 푸시로 대체 (예산 효율화).

---

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 핵심 기능 (Core Features)
1.  **실시간 푸시 알림 (Real-time Push Notification)**
    *   사용자 설정 키워드 기반 알림 발송.
    *   공고 등록 시 지연 없이(Instant) 발송.
2.  **직관적 상세 화면 (Action-Oriented UI)**
    *   **Push to Call**: 화면 중앙에 대형 '전화 걸기' 버튼 배치.
    *   불필요한 정보 최소화, 핵심 정보(임금, 위치, 조건) 강조.
3.  **키워드 구독 설정**
    *   관심 분야(예: 밭일, 사무직, 월세, 빈집 매매) 체크박스 형태 제공.

### 4.2 UI/UX 전략
*   **글자 크기**: 기본 폰트 사이즈 확대 (506070 고려).
*   **Flow**: 앱 실행 → 알림 리스트 → 상세(전화걸기)의 2 Depth 이하 구조.

## 5. 개발 및 운영 전략 (Development & Agile Strategy)

### 5.1 로드맵 (Roadmap)
*   **Phase 1: MVP (최소 기능 제품)**
    *   **Scope**: 일자리/빈집 알림. "알림 → 전화" 기능만 구현.
    *   **Target**: 5070(접근성) & 예비 이주민.
*   **Phase 2: 고도화 (Feedback Loop)**
    *   **Scope**: 사용자 피드백 반영. 상세 필터링(거리순, 세부 직종 등) 도입.
    *   **Action**: 실제 사용 로그 분석 후 UI 개선.
*   **Phase 3: 플랫폼 확장 (Expansion)**
    *   **Scope**: 생활 밀착 정보(농기계, 장날, 보건소)로 콘텐츠 확장. "지역 생활 정보 허브".

### 5.2 민간 주도형 서비스 모델
*   **공공 앱 실패 방지**:
    *   공급자 중심(보여주기식) 지양 → 사용자 중심(철저한 실용성).
    *   '오픈' 후 방치(Zombie App) 방지 → 지속적 업데이트(Agile).
*   **R&R**:
    *   **지자체**: 신뢰도 높은 데이터(일자리/빈집) 제공.
    *   **민간 개발**: 유연한 서비스 기획 및 UX 고도화.

*   3040: 효율의 도구
*   506070: 디지털 복지의 도구
*   이주민: 이주 가이드

가장 적은 비용(Low Cost)으로 **정보 접근성 혁신**과 **인구 유입**을 동시에 달성하는 지역 맞춤형 플랫폼입니다.
